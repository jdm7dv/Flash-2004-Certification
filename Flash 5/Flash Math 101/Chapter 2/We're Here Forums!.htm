Chapter 2 - Angles and Vectors

Where will we draw the line, indeed! Of course, the more appropriate question would be ... how will we draw the line? My research folder is bursting at the seams with examples for solutions to this question. I've been exploring the many source files available to choose the optimal script approach that will provide us with the most efficient code for our continuation of this series' plug-n-play code progression.

This chapter will complete the 'code block' for calculating the distance, setting the angle and drawing the line between two objects using our familiar Flash 4 ActionScript. Bookmark page 3 for your future use, which provides a handy JavaScript Math Object reference for it's built-in properties and methods, as well as a selection of applied functions that utilize these core functions in their equations.

Flash 4 ActionScript

No matter which method is being implemented, the code structure for drawing the line follows these basic steps:

1. Map the coordinates of object #1, whether it is stationary or being dragged.
2. Find the distance of object #2 in relation to these coordinates.
3. Set the angle of the 2 objects on the 2d x-y plane (or 3d x-y-z).
4. Draw the line between these 2 points (or apply to a vector mapping formula)

I must address a very important premise when programming angle in the Flash 4 ActionScript environment. It is standard convention in most computer languages when scripting angle for the unit circle, that the starting point is the positive x axis (horizontal, left-to-right), incrementing counter-clockwise through 360 degrees.

However, Flash does the reverse, incrementing clockwise from the same x axis vector. No doubt this is due to the (0,0) coordinate being positioned in the top left corner, creating a top-to-bottom delineated positive y axis.

Could this be the source of the confusion that persists when learning ActionScript? Even seasoned programmers may falter when attempting to understand this part of the learning curve. We were taught a counter-clockwise approach while sitting attentively in our high school trig class ... remember? Your subconcience does!

Our ultimate goal is to optimize the code with enough accuracy in the computations for the application's correct functionality. Some of the following scripts utilize tables to set variables for use in the various trigonometric functions, which is fine if more accuracy is not essential to the code. However, if we are trying to map very small objects, many objects at once, or iterating coded functions in a loop, accuracy becomes more of an issue in the development of an optimized, error-free script.

Now, to continue with our plug-n-play code progression, let us examine an all-trig function solution to draw the line by Jason Krogh. You often see this code being used by new actionscripters in their copied-and-pasted scripting attempts, due to the popularity of Jason's ActionScript tutorials available at his zincRoe site, and the universal need to set the distance and angle of objects in most programming situations.

You can download the source .fla here: http://www.zincroe.com/flash

Here's the code:

Distance
By: Jason Krogh

Variables - pi, input, x, iterations, i, output, ratio, arctan
Textfields - deltax, deltay, distance, angle
Buttons - MC:circle1 - instance - c1 - 'circle1-button'
MC:circle2 - instance - c2 - 'circle2-button'
Layers - Actions, Graphics, MCs

Frame 1

Comment: =========================
Comment: jason Krogh 1999
Comment: =========================
Set Variable: "pi" = 3.1416 //restricts computations to 4 significant digits-->
Set Variable: "deltax" = GetProperty ( "c2", _x ) - GetProperty ( "c1", _x )
Set Variable: "deltay" = GetProperty ( "c2", _y ) - GetProperty ( "c1", _y)
Comment: =========================
Comment: calculate the distance between the two movie clips
Comment: =========================
Set Variable: "input" = deltax*deltax + deltay*deltay
Call ("sqrt")
Set Variable: "distance" = output //this is our generic distance code from chapter 1-->
Comment: =========================
Comment: calculate the angle between the two movie clips
Comment: =========================
//the following calculations are based on a clockwise progression of the angle, beginning
with 0 degrees at the horizontal x axis vector right (positive value)-->
//Flash increments both x and y axes from the top left corner, so x axis values are positive
increasing to the right and negative increasing to the left, and y axis values are positive
increasing down and negative increasing up-->
//these calculations are expressed in radians, and then converted to degrees by multiplying
the created output by 180/pi. ... 360 degrees = 2*pi rads ... follow this link for a complete
explanation of this radians-to-degrees conversion-->
//each If statement formats the arctangent input to (-1 < x < 1) which returns the result
(-(pi/2) < output < pi/2) displayed by this arctangent graph ... note how both extensions of
//the graph reach to infinity at the output extremes of +/-(pi/2) = +/-(1.5708 approx.)-->
Set Variable: "ratio" = deltay/deltax //initialize axes distance ratio-->
If (deltax > 0) //if x axis vector magnitude is positive [+]-->
      If (ratio > 1) //and if [+]y axis # is larger than [+]x axis #-->
            Set Variable: "input" = 1/ratio //*use the inverse of the ratio to set the right angle
            Call ("arctan") //to the y axis ...arctangent(q) = 1/opposite/adjacent... then set the
            Set Variable: "angle" = 90-output*180/pi //angle output back from [+]y axis-->
      Else If (ratio > 0 and ratio < 1) //or else if [+]x axis is larger than [+]y axis-->
            Set Variable: "input" = ratio //*leave the ratio as is to set the right angle to the
            Call ("arctan") //x axis and to satisfy input for the arctan equation, then set
            Set Variable: "angle" = output*180/pi //the angle forward from [+]x axis-->
      Else If (ratio < 0 and ratio > -1) //or else if [+]x axis # is larger than [-]y axis #-->
            Set Variable: "input" = ratio //*leave the ratio as is to set the right angle to the
            Call ("arctan") //x axis, then set the angle back from [+]x axis by adding the
            Set Variable: "angle" = 360+output*180/pi //[-]output to 360, a full circle-->
      Else If (ratio < -1) //or else if [-]y axis # is greater than [+]x axis #-->
            Set Variable: "input" = 1/ratio //*use the inverse of the ratio to set the right angle
            Call ("arctan") //to the [-]y axis, then set the angle forward from [-]y axis by
            Set Variable: "angle" = 270-output*180/pi //subtracting the [-]output-->
      End If
Else If (deltax < 0) //else if x axis vector magnitude is negative [-]-->
      If (ratio > 1) //and if [-]y axis # is larger than [-]x axis #-->
            Set Variable: "input" = 1/ratio //*use the inverse of the ratio to set the right angle
            Call ("arctan") //to the [-]y axis, then set the angle back from [-]y axis by
            Set Variable: "angle" = 270-output*180/pi //subtracting the [+]output-->
      Else If (ratio > 0 and ratio < 1) //or else if [-]x axis # is larger than [-]y axis #-->
            Set Variable: "input" = ratio //*leave the ratio as is to set the right angle to the
            Call ("arctan") //[-]x axis, then set the angle forward from the [-]x axis by adding
            Set Variable: "angle" = 180+output*180/pi //the [+]output-->
      Else If (ratio < 0 and ratio > -1) //or else if [-]x axis # is larger than [+]y axis #-->
            Set Variable: "input" = ratio //*leave the ratio as is to set the right angle to the
            Call ("arctan") //[-]x axis, then set angle back from [-]x axis by adding
            Set Variable: "angle" = 180+output*180/pi //the [-]output-->
      Else If (ratio < -1) //or else if [+]y axis # is greater than [-]x axis #-->
            Set Variable: "input" = 1/ratio //*use the inverse of the ratio to set right angle to
            Call ("arctan") //[+]y axis, then set angle forward from [+]y axis by subtracting
            Set Variable: "angle" = 90-output*180/pi //the [-]output-->
      End If
Else If (deltax = 0) //else if x axis vector magnitude is 0-->
      If (deltay > 0) //and if there is a [+]y axis value-->
            Set Variable: "angle" = 90 //set the angle at the [+]y axis, or 90 degrees-->
      Else If (deltay < 0) //or else if there is a [-]y axis value-->
            Set Variable: "angle" = -90 //set angle at the [-]y axis, or -90 degrees-->
      Else If (deltay = 0) //or else if y axis magnitude is 0-->
            Set Variable: "angle" = "null" //set angle to null, objects at the same coordinates-->
      End If
End If

Jason's site also has a simplified version for drawing the line that should be considered when your script isn't concerned with the manipulation of values that are generated through the trigonometric function method. This version uses the magic of the Scale function applied to both axes on a movie clip with a 45 degree angle hairline placed in a 100x100 pixel grid. Since each axis is 100 pixels in length, and scaling is measured in percentage, the result is a direct scaling, where each axis' scripted percent of movement becomes 1 pixel in length, thus an accurate scaling of the new line.

Our ActionScript forum has an active member, Stickman, who has provided our community with a drawing the line tutorial for this simplified version. His site also offers many other helpful tutorials that he has accumulated while answering posts to fellow ActionScripters at were-here.

Stickman has an uncanny knack of finding the simple method for scripting, and a study of his work will benefit your further use of mathematics in the Flash environment. The end result we are seeking is efficient code, and his mastery of the tools that Flash has available, applied to the many source files at his site, can replace or embellish the scripted math functions that we presently utilize.

The variables generated using the trig function method are available to the programmer for further use, by placing more objects on the stage to interact with, and calling the parameters for the interactivity once those objects have all been set and their variables initialized.
