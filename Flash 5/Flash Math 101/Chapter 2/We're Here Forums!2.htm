Chapter 2 - Angles and Vectors (Page 2)

Now that Macromedia has decided to include the full function capabilities of the ECMA - 262 compliant JavaScript 1.2 Standard for programming in the Flash 5 scripting environment, our examination of the Flash 4 ActionScript solution that we are familiar with seems redundant. However, there is valuable information available to further comprehend the transition from ActionScript by exploring many other authors' draw-the-line solutions. Since our new focus is Flash 5 scripting, I won't be presenting their arguments, but I will provide the links to their sites or string topic for your own comparison.

Those who are familiar with any programming language, and JavaScript in particular, will find the transition to Flash 5 scripting theory a breeze, and designers who have become familiar with ActionScript will have little problem with the transition as well. But Flash 5's editor-based programming of an obscure Math.method to perform the necessary function won't rationalize the production of the code as well as the defining approach that our pick & axe ActionScript will provide. Once we have explored the principles of the algorithm's coding by dissecting the ActionScript, we'll be better prepared to logically code in Flash 5's JavaScript environment.

Den Ivanov has provided us with our next progression of the code Real Time Triangles which introduces a practical application for our scripting process. His presentation for our next examination is a fully-draggable triangle that can be configured to any triangular shape, with any of the vertices coded to be the main angle opposite to the hypotenuse after the drag operation. The script also dynamically draws the tangent vector of the hypotenuse which connects through this vertex, which creates two right triangles for use in calculating the trigonometric functions, all while the dragging operation is making changes to the triangle's parameters!
You can download the source .fla here.

Whoa! That's the second time that we've seen that 'chicken-wire fence' arctangent script in use. Surely all of those individual multiplications can be avoided by the use of an alternate script? For all of the information about the code that we're planning to replace, including all of the necessary theory that produces the algorithm, follow the links to the Taylor Series and it's derivative theorem, the MacLaurin Series. It's important to know where you've arrived from when deciding where next to journey, so study these 2 links well.

Our friendly tutor at were-here Ken Boucher searched the annals of his harddrive to locate a piece of gold from the archives: a link to the explanation and various degrees of accuracy that can be attained for arctangent through the use of rational approximation.
Programmers' Heaven - Stuchat #36 #37 #38).

The script that is best-suited for our needs is accurate to .01, which should be more than accurate enough for any application in Flash. This formula only works for angles up to 45 degrees, so either an angle-checking script or the large-angle code found at Stuchat will need to be introduced to complete our unit circle scripting.

Arctangent formulation by Ranko Bojanic:

atan (x) = x / (1 + 0.28 * x^2) (|x| <= 1)

The Flash 4 version to include in future scripts:

Set Variable: "x" = input
Set Variable: "arctan" = x / (1 + 0.28 * (x * x))
Set Variable: "output" = arctan

I believe that we can easily conclude that this second code returns the smaller number of compututations, and would be the preferred code to include in our next progression.
