Chapter 1 - Distance (Page 3)

One of the many math-related source files that are available to the Flash community is Pythagorus' Theorem, which accepts input for two sides of a right angle triangle, to return a result for the third side, whether the output is for side a, b or c. The results attained are quite similar to the previous distance script, however there is more structure to the graphics, which calls for a more complex code.

If you have been having trouble following the logic involved when you are coding ActionScript, please take some time here to see how our two previous scripts get utilized in this next application. This may seem a bit elementary at present, but the code illustrates the repetitive steps that are necessary for successful programming.
You can download the source .fla here.

Here's the code:

Pythagorus' Theorem
By: Brandon Williams and Ethan Kennedy

Variables - iterations, x, i, ab, ca, cb
Textfields - a, b, c //include only specified outlines, check numbers, characters-->
              - hidden_a, hidden_b, hidden_c //do not include font outlines-->
Buttons - Symbol 4, Symbol 5, Symbol 6
Layers - Comments, Actions, Graphics

Main Timeline

Comment: Pythagorean theorem: a^2+b^2=c^2
Comment: This app does a lot of things behind the scene so you will need to look everywhere for actions. It is basically 3 text fields, two buttons, and a movie to display error messages. The process goes in 3 easy steps:
1) user enters any number for and two text fields.
2) once the "calculate" button is pressed it checks for errors e.g. not enough variables entered or "c" being greater than "a" or "b".
3) then the operation is performed with given variables.

Actions

frame 1

Comment: Initialize Variables
Comment: the amount of times the square root will repeat (for accuracy only--higher the number the more accurate)
Set Variable: "iterations" = 50 //note the count compared to our average-->
Stop

frame 5 - label - calc_ab

Comment: if you are given the variables "a" and "b" you use the equation: a^2+b^2=c^2
Set Variable: "ab" = (a*a)+(b*b)
Comment: Then you need the square root of (a^2+b^2) so you use Newton's iterative equation.
Set Variable: "x" = 1 //full discussion on page 1-->
Set Variable: "i" = 0
Loop While (i<=iterations)
      Set Variable: "x" = x-((x*x-ab)/(2*x))
      Set Variable: "i" = i+1
End Loop
Set Variable: "c" = x //sets variable c = output-->

frame 6 - label - calc_ac

Comment: if you are given the variables "a" and "c" then you use the equation: c^2-a^2=b^2
Set Variable: "ca" = (c*c)-(a*a)
Comment: Then you need the square root of (a^2+b^2) so you use Newton's iterative equation.
Set Variable: "x" = 1 //full discussion on page 1-->
Set Variable: "i" = 0
Loop While (i<=iterations)
      Set Variable: "x" = x-((x*x-ca)/(2*x))
      Set Variable: "i" = i+1
End Loop
Set Variable: "b" = x //sets variable b = output-->

frame 7 - label - calc_bc

Comment: if you are given the variables "b" and "c" then you use the equation: c^2-b^2=a^2
Set Variable: "cb" = (c*c)-(b*b)
Comment: Then you need the square root of (a^2+b^2) so you use Newton's iterative equation.
Set Variable: "x" = 1 //full discussion on page 1-->
Set Variable: "i" = 0
Loop While (i<=iterations)
      Set Variable: "x" = x-((x*x-cb)/(2*x))
      Set Variable: "i" = i+1
End Loop
Set Variable: "a" = x //sets variable a = output-->

We can follow the script logic easily with this short code, but if we were to program a very complex movie in the same manner, it would soon become an unmanagable, byte intensive monster. That's where object-oriented programming enters to save the day! These repetitive blocks of code can be set in one labelled frame on your actions layer, then you may call the script each time you require it, or it can be running continuously in the background.

My valiant attempt at building a game when I was learning ActionScript is an example of how this complexity can overwhelm the script. When the logic failed and the de-bugging began (pardon the pun!), the uncommented, complex code stood as a roadblock for the successful progression of the task at hand. It's time to dust off the real cobwebs that have invaded the scripting, and evolve the game into an efficient example of using mathematics to code interactivity in the Flash environment. Let's take a look!
