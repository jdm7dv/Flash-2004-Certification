Chapter 1 - Distance (Page 4)

You may have missed our main characters' introduction; either start again here or continue on to the script below!

The Spider vs. The Fly

Welcome to Spidee's web! She's getting very hungry waiting for us to rework the code. She can find her prey easily once the hapless victim has entered her lair, but she can't seem to complete the kill scene ... "after all, a girl's gotta eat!"

Although this 'shortest distance' code functions correctly, we will still begin our journey by reworking the script for a more efficient solution. Presently, each intersection's code sets the new variables, applies these to find each path's distance squared, and compares the results to find the shortest route. This sounds fairly straightforward, unless you're using it 152 times in the movie! More efficient scripting would have each intersection update the variables only, then by using the Call function, have the new data available for input to a short series of distance evaluation scripts.

Also, I have in place a 'cheat' script that sets the coordinates based upon the location of the end of each path that leads from the intersection. These points are then compared for their distance to the Fly utilizing Cartesian coordinate equations, or "as the crow flies" comparison. A real spider spins the spiderweb for travel from point to point, so a more accurate distance evaluation would take into account the entire path, including the initial distance from the intersection in question.

This subtlety isn't important to our Spidee script, however I've mentioned it because your application may demand the more accurate results that can be attained applying code based upon other coordinate systems. Explore the theory involved when calculating the actual path distance, which is commonly referred to as taxicab geometry.

3-Path Intersection

//MC:Liveweb initializes the 75 vectors that make up the spiderweb construction and sets their visibility to False. As each new intersection is reached, the vector that was travelled to get there changes visibility to True, thereby presenting it for use in our still-faulty collision detection script, which is a topic for another discussion in chapter 4-->
Stop
Begin Tell Target ("/Liveweb") //refers MC:Liveweb-Liveweb on the main timeline-->
       Set Property ("v1", Visibility) = True //sets vector1's visibility to true-->
End Tell Target
//(below) the end point of each path leading from an intersection of Spidee's web has a labelled MC (Pos#), that gets mapped here with 2-4 x&y GetProperty snippets, depending upon the number of paths that lead from the intersection. This sets the variables that are put to use in the Pythagorean Theorem code below this-->
Set Variable: "dp1x" = GetProperty ("/Pos5",_x)
Set Variable: "dp1y" = GetProperty ("/Pos5",_y)
Set Variable: "dp2x" = GetProperty ("/Pos2",_x)
Set Variable: "dp2y" = GetProperty ("/Pos2",_y)
Set Variable: "dp3x" = GetProperty ("/Pos0",_x)
Set Variable: "dp3y" = GetProperty ("/Pos0",_y)
Set Variable: "dp4x" = "Null" //null replaces previous output in unused textfield to
Set Variable: "dp4y" = "Null" //avoid confusing previous with present readings-->
Set Variable: "fly_x" = GetProperty ("_level4/Fly_Guy",_x) //sets Fly's location
Set Variable: "fly_y" = GetProperty ("_level4/Fly_Guy",_y) //for input-->
Set Variable: "dist1" = Int(((dp1x-fly_x)*(dp1x-fly_x))+((dp1y-fly_y)*(dp1y-fly_y)))
Set Variable: "dist2" = Int(((dp2x-fly_x)*(dp2x-fly_x))+((dp2y-fly_y)*(dp2y-fly_y)))
Set Variable: "dist3" = Int(((dp3x-fly_x)*(dp3x-fly_x))+((dp3y-fly_y)*(dp3y-fly_y)))
//(above) each of the equations solve for the hypotenuse squared (dist1, dist2, dist3 or dist4 if it's a 4-point intersection) using the input variables produced from the previous lines of code. When comparing relative distances between sets of objects we can forego the use of the squareroot iteration, because each distance's square compares accurately to the other's square-->
If ((dist1<dist2) and (dist1<dist3))
      Go to and Play ("1.2.1.1")
Else
      If ((dist2<dist1) and (dist2<dist3))
           Go to and Play ("1.2.2.1")
      Else
            If ((dist3<dist1) and (dist3<dist2))
                 Go to and Play ("1.1.1.0")
            End If
      End If
End If
//(above) this nest of If statements checks all possibilities to find the shortest distance squared, then sends the main movie to the appropriate position on the timeline, visiting the MC:Pos-Pos# which marks the intersection and whose frame holds the shortest distance script. There are 43 different scripts, each used up to 5 times (eg. Pos1a-Pos1e).I've prepared all three examples of the script to display how the code for each type of intersection is set up. I also tried employing 'Else If' to avoid the nested conditions, but if I recall it wouldn't function. I've since learned how important it is to comment your code to remove this later guesswork!-->
//The webpath label that is called with Go to and Play is derived through this convention: 1x.2x.3x.4x ... 1x (1-4) marks the web's main verticals, from right to left ... 2x (1-12) marks vertical level, from top to bottom ... 3x (1-2) marks horizontal direction, 1-to left, 2-to right ... 4x (0-1) marks vertical direction, 1-for down, 0-for up-->

This code is a typical example of the 152 'shortest distance' scripts in the game. I must restate that I had programmed this code while learning ActionScript, and trying to manage all of these similar scripts tends to become a logistical nightmare.

Keep this in mind as we explore further into this game, because solutions utilizing object-oriented programming in blocks of code can be more effectively implemented, resulting in less programming for you and less work for your surfers' processors.

Your project for submission next chapter, should you accept it, is to convert these 'shortest distance' scripts into 3 blocks: either 2, 3 or 4-path intersections, all calling their respective script using dynamically generated variables. I will post all successful submissions for comparison, and we'll evaluate the efficiency of each script to select the best example.
