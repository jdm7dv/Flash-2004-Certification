Evaluating The _droptarget Property Jessica Speigel
http://www.webstyles.net
preview
download - PC - Mac

If you want to trigger actions based on the _droptarget of a draggable movie clip, this is the tutorial for you! This example will set a variable that contains the _droptarget property and target the movie clip that variable contains. If you are not familiar with drag and drop movie clips, take a look at our Drag-n-Drop Movie Clips tutorial.

Step 1 - Create the movie and symbols

Create a new movie. Next, create a movie clip symbol that has a Stop action in the first frame, this will be the movie clip that gets targeted based on the _droptarget property variable. Now, make one more movie clip symbol, this will be the draggable movie clip. We also need a button to go inside of the draggable clip. Open the draggable clip up for editing and drag the button into it from your library.

Step 2 - Add the actions

Open your draggable clip for editing (if it's not still open) and double click the button to bring up the Instance Properties dialog. Add the following code:

On (Press)
Start Drag ("")
End On
On (Release)
Set Variable: "target" = GetProperty ( "", _droptarget )
If (target ne "")
Begin Tell Target (".." & target)
Go to and Play (2)
End Tell Target
End If
Stop Drag
End On

Don't worry it's not scary, just think about it like this:

When the user presses the button inside the draggable clip
Start the drag operation
Wait until the user releases the mouse
Set a variable called target to the value of the _droptarget property
If the variable target contains a value (i.e. the clip was dropped on another movie clip, or else there won't be anything to target!)
Start a Tell Target action using the variable target as the target
Stop the drag operation.

A couple more notes...

Notice the following two lines:

Set Variable: "target" = GetProperty ( "", _droptarget )

Begin Tell Target (".." & target)

Make sure the properties look like this:

Notice that both lower boxes have the buttons activated, this tells the Flash player that it's looking at an expression rather than a literal. If this button was set on then the variable target would contain "GetProperty("", _droptarget)" and your movie will try to find a target named "".." & target", so make sure that's set correctly as it can cause lots of undue stress.

That's it! I bet it wasn't nearly as hard as you thought.
