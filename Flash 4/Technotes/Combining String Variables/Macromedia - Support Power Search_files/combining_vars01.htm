Combining string variables in Flash

Product: Flash Platform: All Versions: 4 and above ID: 14261

Combining multiple string variables together in a programming language is commonly called "concatenation". This is the process of taking two text strings held in two different variables and combining them into a unique new string.

What is a string?
A string in Flash is a series of alphanumeric characters held in a text field or a variable. Strings can be created in multiple ways:
1

Using Editable Text Fields:
If an editable text field has a variable name in Text Field Properties then the characters inside the field can be accessed as a string.

See the Flash example at the end of this TechNote for an example of string concatenation using editable text fields.

2

Using ActionScript:
String variables can be created with ActionScript without requiring a text field or any user input.

For example, this illustration shows a keyframe script setting a variable named "geographic_location" with the value "Indianapolis". In this script, "Indianapolis" is a string.

Notice how the "abc" button is set to "String Literal". This tells Flash to treat this value exactly as entered.

When is a string not a string?
When it's an expression.

In most programming languages, a value must be "typed" before you can use it. This means the value must be declared to be a string or an integer number or other value.

Because ActionScript is not a "typed" language, this is not necessary in Flash. Setting the "ABC" button to "Expression" will tell Flash to treat the value as a number, or to perform an evaluation of the content of the value.

For example:
10 + 13 as a string literal will give the value "10 + 13".
10 + 13 as an expression will return the value "23".

In more complicated ActionScripts, Flash may interpret a value incorrectly. If an integer is entered when Flash expects a string value then Flash assigns the value "TRUE" to the integer.

If a string is entered where Flash is expecting an integer value, Flash assigns a value equal to the number of characters in the string. So if the entry field was expecting an integer and the user entered "ABC" Flash would use the integer value of "3". However, if Flash is unable to convert the string to a number, it will assign an integer value of zero. For example, "ABC + 3" cannot be interpreted as a number, so it would be evaluated as zero.

If Flash sees a string in ActionScript where it's expecting an integer, it assumes the string is a reference to a variable name, and will look for the value of that variable. If this string is not a declared variable in the movie, it has no value.

Putting it all together
The key to concatenation is understanding the use of the '&' or ampersand character in ActionScript.

The ampersand is used in ActionScript to combine two strings, or combine a string with the value of an expression or a variable.

The following movie combines three strings which can be entered by the user. The variable names are "first", "last" and "age".

The ActionScript attached to the "show combined result" button is shown in this image.

What does it do?
The ActionScript expression in the "Value" field is explained below:

1

Finds the value of the variable 'first'. Because this value is in an editable text field, Flash interprets it as a string.

2

Uses the '&' concatenation operator to add more information after the value returned by the "first" variable.

Adding the " " (which can be read as QUOTE SPACE QUOTE) after the ampersand adds an empty space. In ActionScript, any items between quotes will be treated as a string. In this case, the string is just a single space character.

3 Repeats the process for the 'last' variable.
4 Adds a comma and an empty space after the value of the 'last' string, then adds the value of the 'age' variable.
5 Displays the final string result in the "Result" field.
