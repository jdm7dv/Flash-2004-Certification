With the many interactivity enhancements included in Flash 4, Flash designers will surely be developing more and more sophisticated user interfaces for people to use. With this article, I will demonstrate two techniques now possible with Flash 4: creating draggable components and making a double-click button.

To illustrate these techniques I will use one of the movie clips in the Shockwave.com homepage I created. These "modules," as I like to call them, are reusable components that house different types of content. A visitor to the site can interact with them by clicking and dragging the module by it's handle and placing it wherever they want on the screen, or they may double-click on the handle to close or open it. Creating draggable components
I was working on a demo for Flash 4 with a pre-alpha build in which I was creating an OS-like GUI. After creating a real-life looking 'notepad' application using Flash 4's new editable text fields, I wanted a way to be able to drag the notepad around as if it were a true app. After seeing my demo and listening to my wishes, the engineering team whipped out a drag feature within a few days! Drag features expand greatly the interactive capabilities within a Flash movie, from draggable scrollbars to volume knobs to drag-and-drop shopping cart systems.

Here's how this works. First create the item you wish to drag and make it a movie clip. In my case, the movie clip is the module which essentially has two states: open and closed, as well as the animation frames in between the states. Give the movie clip an Instance name (double-click on the movie clip). Mine, for example, is called "featuremod".

Second, you'll need to define and area inside the movie clip that will be used as the handle. This is what the user will click on to drag the clip, so you may wish to make it appear as something that is clickable or draggable, like a grip of some sorts. The handle should be a button, or it could be a button inside another movie clip (more on this in the next section). Here is my module movie clip with the handle button:


'Draggable movie clip containing handle button

Notice that my handle button is actually invisible, as the grips are part of my original artwork on a separate layer. To do this, I've created a button with only a hit state defined. Flash represents invisible buttons with a semi-transparent light blue item the shape of the hit state.

Next, you'll need to set the actions for the button in the Instance Properties dialog box. Basically, there are two On MouseEvents that you need to assign: "On (Press)" and "On (Release)".


On (Press) and On (Release) actions for setting drag operations

In the "On (Press)" action, select "Drag Movie Clip" from the pull-down action list. "Start Drag Operation" will be selected by default. In the "Target" field, type in the path name of your draggable movie clip. In my case, this is "/featuremod". You may also select your movie clip from the new Target Editor found under "abc" button to the right of the field. This way you will be sure to get the path name accurate. You may also notice the option in this dialog to "Constrain to rectangle." I won't go over this here, but this feature allows you to constrain the user's dragging capabilities to a specified rectangle. (For more information, check out the Flash 4 help system.)

In the "On (Release)" action, select "Drag Movie Clip" from the pull-down action list. Then choose the "Stop Drag Operation" option at the bottom. In this example, this will simply leave the movie clip where the user drops it. In more advanced uses, you might have an action that calculates where it was dropped for drag-and-drop scenarios and do an If-Then action to run different actions accordingly.

At this point, if you preview your movie, you should have a draggable movie clip!
