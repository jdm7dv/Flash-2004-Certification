When did you first pick up Flash?
I first used Flash when it was touted as an animated GIF program, when it was called FutureSplash Animator in early '96. I thought it was a really awkward tool for creating GIFs, but was amazed at what it could do in such a small file size.

I got serious about it when I joined Macromedia in early '97, just after we had acquired FutureWave. At that point, they had just added bitmap and sound support to release Flash 2. I felt that it was then ready to be an incredibly powerful tool for bringing impressive animations to the Web.

What was it about Flash that got the passion flowing?
I think what sparked my creative spirit was seeing some really great Flash animation work created by the in-house design team. Those are some of the early animations that were on the Macromedia site, on the main pages.

As the Flash designer on the Macromedia.com Web team, I had just started working closely with the Flash team. There were a lot of undocumented features and tips that I learned from the team that allowed me to refine my Flash skills, especially in areas like streaming and minimizing file size. I can't overstate how much of a privledge it's been to work with the Flash engineers. I'm like an honorary member of their crack QA team - I've been working with Flash 4 since early alpha builds, pushing it to it's limits and helping them discover a lot of bugs. I think having so much input in how the tool that I use daily developes is incredible.

Back to the future... what's changed from Flash 3 to Flash 4?
I think there are 4 key reasons why Flash 4 is the future of dynamic sites on the Web—first, the UI and usability enhancements, like a new library palette, a removable timeline—especially handy when you're using multiple monitors—improved tool palette, curve manipulation enhancements, a more dynamic publishing system, and the ability to edit a symbol in place, so you can work on symbols in context of the full scene. All these enhancements make Flash 4 much more accessible to beginners, as well as more efficient for experienced users.

Second, interactivity and scripting capabilities—for example, you can have an editable text field, so that users can input text into Flash movies, making it possible to do Web forms in Flash—also logic commands, variables, and the ability to import variables from text files or URL strings.

Third, support for the MP3 compression format—this not only adheres to industry standards, but it also cuts the file size of sound in Flash by at least half.

Fourth, Flash file format acceptance and player ubiquity—with the 2 major browsers shipping with Flash, more and more people have Flash, and I think that's one factor that will push Flash into the main stage.

And Free Source!
Yeah... the Flash 4 player source will be freely licensable, providing a way for systems like Linux to incorporate Flash playback, even at the system level.

I love that... to the system level. Where to you see Flash and the Web going from here?
In the future, with the player being freely available to license, and the acceptance level already over 80%, it's only a matter of time before you stop seeing 'Get Flash' buttons.

I think that Flash 4 will see a lot of entertainment-oriented use with its expanded interactivity, but I also feel that there will be a shift from entertainment sites to standard commercial sites exploiting the capabilities and features that Flash 4 offers. I think that there will be less reliance on HTML standards, including dynamic HTML—I think it's clear that there's difficulty coming up with concensous between Netscape, Microsoft, and all the other interested parties, and Flash already has a solution that's widely accepted.

Welcome to the next level... to entertainment and beyond!
