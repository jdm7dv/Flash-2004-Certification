Introduction

So, from CGI to ASP. The transition couldn. t be more different. Essentially we move from one operating system to another. You know, kind of like from Mac to PC. As with the Mac . PC battle, there will emerge two "factions". Both of which will vigorously defend their corner. But whatever people say, ASP is a language with an easy learning curve, unlike PHP3 (in my opinion, of course).

Down to business

Anyway, enough of the pre-amble, lets get down to business. In the script you will find two functions which you should already recognise. These being URLEncode(ByVal s) and HexChar(ByVal i). If you don. t already know, what these functions do are to encode the results in such a way, which allows Flash to read them. The alien object to you will possible be Set SMTPObj = CreateObject("WebHostMeMail.SMTP"), this is actually a precursor to open up an SMTP connection (SMTP being the outgoing mail host). This is where you come in... you will need ask the web host administrators what type of mailing system you use, or have access to. You could either use just a normal SMTP host, or they can provide you with a third party system, such as ASPMail, or its free equivalent, JMail. Whatever, but it is here where you could possibly come "unstuck". Click here to find some example code which could be used as a replacement.

By now you should have established the Mailhub... no, OK then, just get a free host account with WebHostMe, which is the service I am using for this example. With WebHostMe*, you can get an account to host your ASP scripts, and just call them from your other host.

So, we. ve established the complicated stuff. Now all you have to do is create so kick arse Flash movie. The required fields are . name, email, subject and body. If you look at the example I have provided, you will see all these fields. You will also see a send button which, when pressed, transmits the results to the ASP script. The button action looks like this .

In Flash movie

In Flash movie
On (Press)
     Load Variables ("/mailform.asp", "", vars=POST)
End On
On (Release)
     Go to and Stop ("start")
End On

What's that you say, "Huh!?". To explain simply, when the button is pressed, it loads the ASP script, which in turn requests the variables name, email, etc. With the variables now in the ASP script, it sets up a connection with the Mailhub (Mail server), formulates the variables, and sends them. The thank you result is then sent back to the key frame start, in the Flash movie .

In ASP script

result = SMTPObj.EMail
message = "Message Sent " & result

The added complexity is the &confirm=2. This sets the variable confirm which prevents the BACK button being displayed. Now I haven't yet discussed the back button, but if you look at the example provided, there exists a movie-clip called back. It has three stages .

Nothing displayed - Back button displayed - Home/Nothing displayed (Send confirmed)

On the key frame start you will find an If/then/else statement which checks the state of confirm .

In Flash Movie

If (confirm = 1)
      Begin Tell Target ("/back")
            Go to and Stop ("2")
      End Tell Target
Else If (confirm = 2)
      Begin Tell Target ("/back") 
            Go to and Stop ("3")
      End Tell Target
End 
If Stop

The key to this If/then/else statement lies in the ASP. You can do the variable checking in the Flash movie, but as I have already said, it is wise to move all load to the server, so the client can have a better and faster experience. The other reason for putting the variable checking in the ASP script is to help you with more complicated objects later on. For example strenuous mathematics, where sin/cos/tan can be performed outside and just passed in.

The F4 Movie

NUMBER to easy follow steps.

1. Create 4 (four) text field boxes (the ones you can write in!). Name all four boxes using the names . email, name, subject and body. In the body text field, set the properties as so - ALLOW multiline - and - ALLOW word-wrap. Also make this box bigger than the others, ie - more lines (rows).

2. Create a new button (Ctrl+F8) called send and decorate as wanted, but remember to put the word SEND somewhere. Give it the following actions . 

On (Press)
     Load Variables ("/mailform.asp", "", vars=POST)
End On
On (Release)
     Go to and Stop ("stage2")
End On

(send using POST, and click on target, but leave it blank)

NOTE: We haven't yet created any key frames yet, so just type in the names.

3. Assign the key frame you are working on with the action . STOP. Label it Stage1.Create a new blank key frame . Assign it the action . STOP, and label it Stage2.

4. In this new key frame, add a text field. Set its properties as so - ALLOW word-wrap - DISABLE editing - DISABLE selection - Call it message.

 

 

5. Create a new movie clip (Ctrl+F8) and call it back. Also give it the instance name back.

6. In the movie clip back create three blank key frames. Give each the action - STOP.

7. Give the first key frame the label 1 (one), and go up sequentially from there. ie - Frame1 - One Frame2 - Two Frame3 - Three

8. In the second key frame or the movie clip back, create a new button. This will be our back button, which will return the user to Stage1. Put some graphic there, or text saying BACK, and change all the different stages.

9. Return to the movie clip back and click on key frame two, labelled 2. There now should be a back button on it. Double click on the button, and then click on the Actions tab. Assign the back button the following actions - 

On (Release)
      Begin Tell Target ("../")
            Go to and Stop ("stage1")
      End Tell Target
End On

NOTE: To explain the following actions you should understand Tell Targets. When the back button is displayed you want the user to be able to go back to the place where they first entered the data
