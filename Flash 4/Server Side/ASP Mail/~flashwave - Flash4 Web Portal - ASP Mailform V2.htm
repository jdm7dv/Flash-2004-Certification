Introduction

Well this is a re-release of the popular first version of the ASP mailform. This edition adds some cool new features as well as optimising the features already there. The thing which has changed the most is the modulisation of the whole script. This increases the overall processing speed of script. Another thing to note is the use of cookies to store the users name and email, so if they every wish to send some more comments to you, then they wont have to re-enter their details.

V2 also adds a simple encryption and decryption routine which work in conjunction with the cookies. Although the encryption constitutes nothing more than moving the ASCII value up 10 places it will still go towards providing piece of mind to the user.

(join the mailing list to get the latest code and updates).

There are now two main functions in this script. The first one has to be called just before the form is displayed. This is because it gathers all the information from the cookies and returns the strings to the F4 form. You have to call it just before the form is displayed because of server lag and processing time. If you called it right on the form, then possibly, when the user was entering his/her details they would, if the cookie was empty, disappear. Of course you don. t have to utilise the cookie functions, and that is fine, because the script will function perfectly well without them.

To use cookies or not to use cookies!?

It is a difficult one... many people believe cookies can be vary dangerous objects of code. Ones so dangerous that they employed computer programmers to develop software to kill them. But what is a cookie? To answer simply, a cookie is a small piece of text, yes plain text, which holds information, such as a name, a number, etc. Cookies can hold anything, even a password, but they generally don. t. It is a myth that cookies can wreak havoc on your hard drive, they cannot read information from your hard drive, nor can they transmit details about your family, your car registration, your computer serial number, etc. The only possibly, and only conceivable way they could do this, is IF YOU HAVE ALREADY ENTERED THESE DETAILS ON A WEB SITE, and even then, only the person who sent them can receive them. This is due to cookies storing the domain IP address of the sender. All the same, you should provide your users with a notice explaining the situation with the cookies, and explaining that they will be encrypted.

Still no cookie?

Always remember to take a look at the F4 movie that accompanies this tutorial. As always this could get a little sticky. To start with, if you are not using cookie then you will not need this line at the start of your movie.

            Load Variables ("http://yourhost/mailform2.asp?status=getcookienames&EncryptCookies=no", 
            "", vars=GET)

("" = Target which has nothing in)

What this does is to load the cookie values into the movie.

Remember I am relaying on you having read the first tutorial so I don. t have to re-explain the details of the text fields here. I am also going to explain the movie construction in little detail but using my source as a framework. If you have problems with the simple parts of this movie, like for instance creating text entry fields, please refer to this.

  1. At Stage one you will need to add another text entry field called "date". Make it quite large because it houses the details of the last form submittal. Disable selection and edit.
  2. Still at Stage one, create a small box, and turn it into a button (Ctrl-F8), call it "Cookies Allowed Button". Now using the same button turn it into a movie (Ctrl-F8) and call it "Cookies Allowed Movie".
  3. Edit this movie by double clicking on it and selecting "Edit Symbol".
  4. Go to the first frame and label it "1", and give it the action "Stop". Now add another key frame. Give the second key frame the label "2" and the action "Stop".
  5. What you should now have is two key frames with an identical button in each.
  6. Edit the first button in the first key frame and then onto the "Actions" tab. Give it the following action . 
            On (Release)
            Set Variable: "../:option1" = "&HateCookies=no"
            Go to Next Frame
            End On
  1. Now click on the second key frame and edit the button finally ending up on the "Actions" tab. Give the second button the following action . 
                  On (Release)
              Set Variable: "../:option1" = "&HateCookies=yes"
              Go to Next Frame
              End On
  2. Still on this second key frame, make an "X" or a tick, or something to show that the box has been pressed, and add it into the centre of the button.
  3. What does this all do? Well simple, it gives the user the choice between their cookies being encrypted, or not. By default the cookies are not used, but when they press the button that changes.
  4. We are now going to create another button on the main stage using the same instructions (num2 . num8). This time though, the button will be called "Cookie Encryption Button" and the movie, "Cookie Encryption Movie".
  5. Also give each button the following actions respectively . 
                  On (Release)
              Set Variable: "../:option2" = "&encryptit=yes"
              Go to Next Frame
              End On
              On (Release)
              Set Variable: "../:option2" = "&encryptit=no"
              Go to Next Frame
              End On
  6. What does this one do then? This second button also provides the user with the option of having their cookies encrypted.
  7. Now we move onto the Send button which submits the users details to the ASP script. This button is found on the main stage.
  8. Edit the button and you should find some actions already listed. You will need to remove all these actions and add these ones. 
                  On (Press)
              Load Variables ("http://yourhost/mailform2.asp?STATUS=sendingmessage" 
              & option1 & option2, "", vars=POST)
              End On
              On (Release)
              Go to and Stop ("stage2")
              End On
  9. With V2, instead of using a "String Literal" with the "Load Variables" option we are using an "Expression", because this allows us to add the extra "Query String" options on the end of the URL. Also the target button has to be pressed, and left empty.

That's it! The rest was/is explained in the previous tutorial for V1.
