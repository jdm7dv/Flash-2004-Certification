Continuous Feedback Buttons

by Bill Tagliaferro
[Download Source]

This tutorial is designed to show you how to build what I call continuous feedback buttons. Continuous feedback buttons are simply buttons that as long as you press them or have your mouse over them they continue to provide actions. An example might be a scroll arrow as you would find on your browser window. As long as you continue to press the scroll arrow the window continues to scroll up or down.

This tutorial is intended for beginner Flash 4 users.

  • No previous tutorials are required.
  • Download the source files and have them unzipped for review during this tutorial.

Step-by-step

Overview:

Continuous feedback buttons (CFB) have numerous applications, one of which was mentioned above. Another application might be similar to the one used in this tutorial where the CFB magnifies an image. The whole process can be summarized as follows:

  1. Definition of Variables or Actions.

  2. Building a contineous feedback loops.

  3. Assigning button actions for the loop.

Background

CFB's can be built in a number of ways. It's more important to understand the theory behind a CFB than the mechanics of building one. Basically, a continuous feedback loop in what programmers call an "infinite loop" in the sense that nothing within the loop itself stop it once it has started. What save this loop from being truely an infinite loop is a button. The button can start the loop and then interupt it to stop.



  1. Definition of Variables or Actions.

    As with any good Flash movie planning starts first. Think about your CFB application. What is the CFB going to control, variables or actions? Variables can be assigned to specific instances of movie clips or they can be global. For example, in this tutorial the image that is magnified is named "mouthman" and I have assigned a variable to it called "scaleval". The CFB's (i.e. the "+" and "-" buttons) in this tutorial increas or decrease "scaleval" which in turn becomes the Scale Property for "mouthman".

    The point is this, pick variables names that relate to what the CFB will do. Then descide whether the variable should be global or assigned to a specific instance of a movie clip. In this tutorial we will assign the variables to the "mouthman" movie clip.

    One last point. There can be many application where variables are not needed for a CFB. An example might be where your actions just move a movie clip to the next or previous frame.

    1. The first step is to define variables and assign them to your movie clip. In this case I want to scale an instance of mouthman to simulate magnification. The variable I will use to keep track of the curent scale value is "scaleval".
    2. At this point I have an instance of mouthman on stage, that is named "mouthman", and two buttons, one for "+" and the other for "-".
    3. To define a variable all you have to do is set it. In this case we want to define a variable of "mouthman" as "scaleval" and initially set it to 100 (for 100%).
    4. Double click on the first keyframe and assign the following actions:

      Set Variable: "scaleval" = 100


  2. Building a Continuous Feedback Loop.
    1. Now create another new movie clip (CTRL + F8). Name this clip "Button + Trigger".
    2. This movie clip will have three frames all of which are keyframes, so insert two more keyframes (F6).
    3. Double click on the first keyframe and give it an action of "Stop".
    4. Double click on the third keyframe and give it an action of "Go to and Play (2)"
    5. Double click on the second keyframe to open its Frame Properties window and click on the Action Tab.
    6. Assign the following action to this frame:

      Set Variable: "/mouthman:scaleval" = /mouthman:scaleval + 10
      Set Property ("/mouthman", X Scale) = /mouthman:scaleval
      Set Property ("/mouthman", Y Scale) = /mouthman:scaleval

    7. Go back to your main movie and drop an instance of this clip on stage.
    8. Give it the name "uptrigger".

    9. Now create another new movie clip (CTRL + F8). Name this clip "Button - Trigger".
    10. This movie clip will have three frames all of which are keyframes, so insert two more keyframes (F6).
    11. Double click on the first keyframe and give it an action of "Stop".
    12. Double click on the third keyframe and give it an action of "Go to and Play (2)"
    13. Double click on the second keyframe to open its Frame Properties window and click on the Action Tab.
    14. Assign the following action to this frame:

      Set Variable: "/mouthman:scaleval" = /mouthman:scaleval - 10
      Set Property ("/mouthman", X Scale) = /mouthman:scaleval
      Set Property ("/mouthman", Y Scale) = /mouthman:scaleval

    15. Go back to your main movie and drop an instance of this clip on stage.
    16. Give it the name "downtrigger".


  3. Assigning Button Actions.
    1. Drop two instances of any button into your scene.
    2. Add a "+" label one and a "-" label to the other.
    3. Double click on the the "+" button and add these actions:

      On (Press)
      Begin Tell Target ("/uptrigger")
      Go to and Play (2)
       End Tell Target
       End On
      On (Release)
      Begin Tell Target ("/uptrigger")
      Go to and Stop (1)
       End Tell Target
       End On

    4. Double click on the the "-" button and add these actions:

      On (Press)
      Begin Tell Target ("/downtrigger")
      Go to and Play (2)
       End Tell Target
       End On
      On (Release)
      Begin Tell Target ("/downtrigger")
      Go to and Stop (1)
       End Tell Target
       End On

Troubleshooting

The main problem you will have in this tutorial is remembering the paths to movie clips and variables. Try to use names that you will remember and have some association with the movie, instance or variable.
