FlashZone: Tutorials: Using the TimeTrigger Component

Using the TimeTrigger Component

by David J. Emberton
[Download Source]

Oftentimes, its necessary to invoke Flash actions at precise intervals, or just long intervals. In the former case, varying frame rates on different systems can upset your sychronization and timing. In the latter case, its imprecise and inconvenient to insert endless numbers of blank frames. So, what to do? By taking advantage of the GetTimer function, you can use your computer's clock to trigger events in your movie. To make the task easier, I've put together a prebuilt component called TimeTrigger. Flexible and easy to configure, TimeTrigger should be useful any time you need to trigger single timed events or even a series of them.

To complete this tutorial on implementing TimeTrigger, you should have a working knowledge of creating symbol instances, and editing ActionScript statements.

Get the Goods

  1. Click here to download the TimeTrigger source file.
  2. Open up a new Flash 4 document and use the File > Open as Library command to open timetrigger.fla as an external Library.

A Sample Scenario

  1. In this exercise, we'll be creating a fake scenario to demonstrate how TimeTrigger might be implemented. Start by dragging your instance of TimeTrigger onto the stage of your movie. In more realistic circumstances you'd place TimeTrigger offstage where it wouldn't be visible (or edit out the graphic content to make it invisible).
  2. Double-click the TimeTrigger instance you just created to edit its Instance Properties. In the Instance Name field, type TimeTrigger - this will give the instance a Target address of /TimeTrigger that you can use in various ActionScript statements.
  3. Without closing the Instance Properties dialog box, click the Edit Symbol button to the left to edit your local copy of TimeTrigger. If you already closed the dialog, just right click (or Ctrl click on MacOS) the instance and select Edit or Edit in Place.
  4. You'll see a number of labeled keyframes within the TimeTrigger symbol. We're chiefly concerned with the first one, Initialize. This is where the configuration variables are located. I won't be detailing the exact contents of the other labeled keyframes, but if you're looking for a challenge please feel free to investigate the coding behind them.
  5. Double click the Initialize keyframe to open its Frame Properties. If you click the Actions tab, you'll see a list of configuration variables for you to set (shown below):

    Comment: Interval in Seconds
    Set Variable: "Interval" = 0
    Comment: Wait for Prompt? True / False
    Set Variable: "Prompt" = True
    Comment: Loops (0 for infinite)
    Set Variable: "Loops" = 0
    Comment: Target Frame to Call (Full target path required, e.g. /MovieClipA/:KeyframeB)
    Set Variable: "Target" = ""

    For clarity, here are definitions of the variables you can change.

    Interval: The number of seconds you want to wait before the desired event is triggered. This needn't be limited to whole numbers, but fraction of a second accuracy is limited by the frame rate you choose for your movie.
    Prompt: Should the timer start immediately, or should TimeTrigger wait to be prompted into action? This will be either True or False, depending on the situation at hand.
    Loops: The number of times you want the timer (and the event) to repeat. Each loop is separated by the number of seconds specified by Interval. If you only want the event to occur once, change this value to 1. For infinite loops, set it to 0.
    Target: The Target address of a labeled keyframe containing the actions you want executed. When the time specified by Interval has elapsed, this keyframe will be called (ie, the "event").

  6. In our sample scenario, we'll be triggering the keyframe /:SetText to be called 5 times, once every 0.5 seconds. We'll also set TimeTrigger to wait for prompting before starting the timer. To do this, set the variables contained in Initialize to the following values:

    Interval - 0.5
    Prompt - True
    Loops - 5
    Target - "/:SetText"
  7. Exit symbol editing mode for TimeTrigger and return to the main Timeline. TimeTrigger is now configured and ready for action!

Completing the Picture

  1. Well, TimeTrigger is now set up - all we need now is to create the supporting elements. In our case, that's the SetText keyframe, a text field and a button to serve as the "prompting mechanism." In other words, the timer won't start until you click the prompt button.
  2. Start by assigning a Stop action to the first keyframe of the main Timeline. Then insert a second (blank) keyframe and label it SetText.
  3. Assign the following action in SetText's Frame Properties dialog box:

    Set Variable: "/:Number" = /:Number + 1

    This is the sample action that makes up our triggered event. In a real situation this would most likely be more complicated, and do something practical like advance a slide show.
  4. Move the playhead back to frame 1 on the main Timeline. SetText will only be called, not actually displayed and thus we want all the symbols to be in frame 1.
  5. Select the Text Tool and click the Text Field modifier. Draw out a text field, then open its Text Field Properties and set its variable name to /:Number. Notice that this is the variable being manipulated in the SetText script.
  6. Lastly, we need a prompt button. Any button will do, you can create your own or grab one from the Buttons Library (available from the Libraries menu). Create an instance of your button, then double click it to open its Instance Properties.
  7. Click the Actions tab, and assign this statement:

    Call ("/TimeTrigger/:Prompt")

    This is the code you'll need to use whenever you're prompting the TimeTrigger timer to begin. You can assign it to a keyframe or button, depending on your interactivity requirements.
  8. Test your movie! When you click the prompt button, the value of the text field should increase to 5 in about 3 seconds. For troubleshooting, see the fully operational timetrigger_sample.fla file contained in the source zip file.

Modifications and Extensions

The methods I've outlined in this tutorial will limit you to one version of TimeTrigger running at the same time. This is because you are setting the configuration variables in the master symbol, rather than an independent instance. You can reuse TimeTrigger in the same movie with different settings using the Set Variable action to alter all the variables, but this won't work if you need two simultaneous timers. In that case the thing to do is have multiple instances of the TimeTrigger symbol, and rather than hard-coding the configuration values into the parent symbol, just use Set Variable actions to set up each individual instance. Just remember to give your instances unique Instance Names!
